## 2. 最初需求

目标是开发一个微信机器人，为私域客户每日推荐精品产品，并尽快跑通 MVP。

- 初期测试垂类：饰品、玩具。
- 推荐权重来源：TikTok、Instagram、小红书近 3 日热点；电商平台近 7 日销量快速增长商品；国际时尚大会和杂志热款。
- 私域群内容：产品来源、采购链接、推荐理由、火热指数等较完整信息。
- 公域内容：产品和精简营销文案。
- 微信机器人框架：优先参考并接入 AstrBot。
- 推荐呈现：一图流产品卡片，随后发送产品链接；产品主图是主要视觉信息。
- 工程原则：参考 GitHub 成熟项目，采用模块化拼接，避免从头重复开发。
- 数据原则：社媒和电商数据优先复用现有开源项目、官方 API、联盟 API 或合规第三方接口。
- 界面要求：中文运营 UI。

## 3. 角色与工作方式

项目按三个角色推进，相关成果已沉淀为文档和测试：

1. 产品需求负责人：明确 MVP 边界、数据字段、推荐流程和验收标准。
2. 代码执行人：完成采集适配、评分、推荐、卡片、管理端和 AstrBot 接入骨架。
3. 测试与审核人：补充自动化测试、风控检查、浏览器验证和阶段验收记录。

对应文档位于 `outputs/01` 至 `outputs/12`。

## 4. 关键产品决策

### 推荐展示

- 微信群中优先发送“一张产品推荐卡片图片 + 一个产品链接”。
- 复杂字段保留在后台、详情指令或私域详细内容中，避免群消息过长。
- 公域文案单独生成，内容更短，更适合直接发布或复制。

### 微信账号测试

- 业务模块开发阶段不需要立即提供微信账号，可先使用 dry-run 和 AstrBot 适配层验证。
- 最终真实群聊联调需要可用于测试的微信账号、测试群以及 AstrBot 对应的微信接入能力。
- 正式联调前保持 `dry_run=true`，防止误发消息。

### 多多进宝接入

- 多多进宝负责商品搜索、推广链接、推广位以及后续订单/佣金相关能力，不负责社媒热点。
- 最终接入通常需要 `client_id`、`client_secret` 和推广位 `PID`。
- 授权信息不写死在程序里，也不提交 GitHub；当前程序只保存环境变量名称和配置状态。
- 用户已经注册多多进宝，但真实凭证尚未写入项目，计划后续通过安全配置界面接入。

### 数据源复用

当前工程预留统一采集器接口。参考方向包括 TikTok-Api、Instaloader、MediaCrawler、Crawlee、Scrapy、官方开放平台和联盟 API。复用时仍需检查许可证、平台条款、登录要求、频率限制和数据合规性。

## 5. 已完成的 MVP 能力

- CSV 商品和趋势信号导入。
- JSONL 外部信号桥接，可承接社媒、电商和时尚趋势数据。
- 统一 `CollectorAdapter` 采集接口和数据源注册机制。
- 社媒 3 天、电商 7 天、时尚 30 天的信号有效窗口。
- 规则化火热指数 `HotScore` 和 Top N 推荐池。
- 饰品、玩具分类推荐。
- 一图流 PNG 推荐卡片生成。
- 产品链接 dry-run 推送。
- 私域详情文案和公域精简文案。
- 商品人工批准、撤销和自动推送前审核。
- 推荐历史、14 天新鲜度控制和每日防重复推送。
- 好、一般、差三档运营反馈。
- APScheduler 每日自动推送骨架。
- AstrBot 插件入口和管理员指令。
- 本地 FastAPI 管理接口。
- 中文运营管理界面，包含候选、审核、数据源、反馈和推送记录。
- 凭证仅通过环境变量引用，接口不返回密钥真实值。

## 8. 新电脑恢复步骤

以下命令以 Windows PowerShell 为例。

```powershell
git clone https://github.com/zhangliang322/recommend-bot.git
cd recommend-bot
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -e ".[dev]"
```

创建本地环境文件时参考 `.env.example`，不要将真实值提交到 GitHub。至少可配置：

```text
PRODUCT_RECO_PROJECT_ROOT=<新电脑上的项目绝对路径>
PRODUCT_RECO_STATE_DIR=<AstrBot 插件数据目录>
PRODUCT_RECO_ADMIN_API_KEY=<随机长字符串>
PDD_CLIENT_ID=<后续填写>
PDD_CLIENT_SECRET=<后续填写>
PDD_PID=<后续填写>
```

运行自动化测试：

```powershell
pytest
ruff check .
```

运行本地演示：

```powershell
$env:PYTHONPATH="src"
python -m product_reco_bot.cli run-demo
```

启动中文运营管理界面：

```powershell
reco-admin
```

一图流卡片默认生成在 `work/generated_cards/`，该目录属于本地运行产物，不提交 GitHub。

## 9. AstrBot 接入步骤

1. 在 AstrBot 使用的 Python 环境中执行 `pip install -e <项目根目录>`。
2. 将 `integrations/astrbot_plugin_product_reco` 放入 AstrBot 的 `data/plugins/`。
3. 设置 `PRODUCT_RECO_PROJECT_ROOT` 和 `PRODUCT_RECO_STATE_DIR`。
4. 保持 dry-run，在测试群执行 `/登记推荐群`、`/待审商品` 和 `/今日精品`。
5. 确认审核、卡片图片和产品链接正常后，再考虑开启真实发送。
6. 自动推送前，将 `config/settings.yaml` 中 `auto_push_enabled` 改为 `true`，并在完成真实通道验证后将 `dry_run` 改为 `false`。
