# 外部数据源桥接阶段交付

更新时间：2026-07-13

## 本阶段成果

- 建立统一外部信号模型，支持社媒、电商、时尚资讯三类信号。
- 建立 JSONL 导入桥接层，可接收独立采集工具和第三方 API 的输出。
- 已兼容常见字段别名，例如 `platform`、`hashtag`、`note_title`、`liked_count`、`comment_count`、`share_count`、`publish_time`。
- 支持通过 `product_id` 精确关联商品，也支持用关键词匹配商品名称、描述和趋势标签。
- 外部信号可以更新浏览、点赞、评论、分享、七日销量、销量增幅、榜单名次和时尚关键词。
- 推荐命令行和 AstrBot 插件均已切换到统一数据源工厂，无需分别维护采集逻辑。

## 时间窗口

| 信号 | 默认有效期 | 用途 |
| --- | ---: | --- |
| TikTok、Instagram、小红书等社媒热点 | 3 天 | 捕捉短周期内容热度 |
| 电商销量和榜单变化 | 7 天 | 判断商品增长速度 |
| 时尚媒体和大会趋势 | 30 天 | 补充风格与趋势标签 |

过期信号会被自动忽略，不会继续抬高当天火热指数。

## 开源项目拼接方式

- MediaCrawler：采集公开小红书内容后，输出或转换为 JSONL。
- TikTok-Api：采集热视频、话题和互动指标后写入 JSONL。
- Instaloader：读取公开 Instagram 内容元数据后写入 JSONL。
- 电商联盟 API：把商品销量、增幅、榜单和采购链接写入商品 CSV 与信号 JSONL。
- 采集工具与推荐服务保持进程隔离，任何单一平台工具失效时都可以独立替换。

这种方式复用了成熟采集项目，同时避免把平台登录、浏览器环境和反爬依赖塞进机器人进程。

## 当前边界

当前已打通“采集项目输出到推荐评分”的技术链路，但尚未配置 TikTok、小红书、Instagram 或电商平台的真实账号、Token、联盟资质和代理环境。示例信号用于验证结构和评分，不代表真实线上数据。

## 下一节点

下一步应选择一个真实数据提供方进行凭证接入。优先推荐电商联盟或供应链接口，因为它能同时提供真实产品主图、价格和采购链接；随后再选择 MediaCrawler 或合规第三方 API 接入一个社媒热点来源。账号授权由运营人员在本地配置，不在聊天或代码仓库中传递密码、验证码和 Cookie。
