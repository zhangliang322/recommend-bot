# 管理接口与数据源配置阶段交付

更新时间：2026-07-14

## 本阶段成果

- 建立本地管理 API，作为后续运营界面的统一后端。
- 提供数据源状态、候选商品、审核、反馈统计和推送历史接口。
- 内置外部 JSONL、多多进宝、TikTok、Instagram、小红书五类数据源配置。
- 多多进宝默认关闭，未配置完整凭证时禁止启用。
- 管理服务默认只监听 `127.0.0.1:8765`。
- 支持通过 `PRODUCT_RECO_ADMIN_API_KEY` 启用 API Key 校验。

## 凭证安全设计

`config/data_sources.yaml` 只记录环境变量名称，例如：

```text
client_id     -> PDD_CLIENT_ID
client_secret -> PDD_CLIENT_SECRET
pid           -> PDD_PID
```

实际凭证不写入源码、不写入 YAML、不通过管理接口返回。状态接口只会显示是否配置完成，以及缺少 `client_id`、`client_secret` 或 `pid` 中的哪一项。

后续图形界面可以继续复用这一模型，并把环境变量提供器替换为系统凭据库或专用 Secret Manager。

## 管理能力

| 接口能力 | 用途 |
| --- | --- |
| 数据源列表 | 查看启用状态、连接准备度和能力范围 |
| 数据源开关 | 启用或停用采集来源 |
| 连接测试 | 检查凭证是否齐全，不泄露凭证内容 |
| 候选商品 | 查看火热指数、标签、采购链接和批准状态 |
| 商品审核 | 批准或撤销自动推送资格 |
| 反馈统计 | 查看好、一般、差的数量 |
| 推送记录 | 查看每个商品最近一次正式发送时间 |

## 验证结果

- 自动化测试：25 项全部通过。
- Ruff 代码规范检查：全部通过。
- 管理服务真实启动：通过。
- `GET /health`：返回正常。
- 数据源接口：成功识别 5 个来源。
- 推荐接口：成功返回排序后的候选商品，最高分约 91.5。
- 安全测试：配置 API Key 后，未携带正确请求头会返回 401。
