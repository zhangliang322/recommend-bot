# 微信私域精品推荐机器人 MVP 测试验收文档

## 1. 测试范围

### 1.1 业务范围

- 微信机器人面向私域客户每日推荐精品产品。
- MVP 垂类限定为饰品和玩具。
- 推荐渠道区分私域群和公域内容。
- 私域群优先输出一图流推荐卡片 + 产品链接，公域输出一图流卡片或简短文案。

### 1.2 系统范围

- 商品数据导入与清洗。
- 热点 / 趋势识别。
- 推荐排序与权重计算。
- 私域和公域内容生成。
- 微信机器人发送。
- 频率控制与防刷屏。
- 审核与日志留存。
- 异常降级与人工干预。

### 1.3 机器人测试重点

- 插件 / 模块配置是否正确。
- 群聊、私聊、关键词触发是否稳定。
- 权限、白名单、黑名单是否生效。
- 消息发送失败、限流、重试、异常日志是否可追踪。
- 第三方模块升级后，核心评分、推送、卡片生成流程是否仍兼容。

## 2. 核心测试用例

### 2.1 商品推荐准确性

| 用例 | 测试点 | 预期结果 |
| --- | --- | --- |
| 饰品热款推荐 | 输入近期饰品热点数据 | 推荐结果包含饰品类商品，推荐理由与热点相关 |
| 玩具热款推荐 | 输入近期玩具销量增长数据 | 推荐结果包含玩具类商品，不混入无关品类 |
| 多来源权重排序 | 同一商品同时命中社媒、电商、杂志趋势 | 排名高于只命中单一来源的商品 |
| 过期热点过滤 | 输入超过 3 日社媒热点 | 不作为高权重推荐依据 |
| 过期销量过滤 | 输入超过 7 日销量增长数据 | 不作为核心推荐依据 |
| 重复商品过滤 | 输入同款不同标题商品 | 推荐池只保留最优商品或合并信号 |

### 2.2 内容生成

| 用例 | 测试点 | 预期结果 |
| --- | --- | --- |
| 私域详细推荐 | 生成群内推荐文案 | 包含商品名称、卖点、趋势依据、适合人群、采购建议 |
| 公域简短推荐 | 生成公域推荐文案 | 文案简洁，不暴露采购链接和内部评分 |
| 一图流卡片生成 | 输入产品主图和推荐字段 | 成功生成清晰 PNG，包含产品名称、主图、火热指数、卖点、来源 |
| 图片生成失败降级 | 产品主图不可访问或渲染失败 | 降级发送原产品主图、产品链接和短说明 |
| 饰品文案风格 | 饰品类推荐 | 语气偏精致、审美、搭配场景 |
| 玩具文案风格 | 玩具类推荐 | 语气偏趣味、安全、收藏或亲子场景 |
| 避免夸大宣传 | 商品只有普通销量增长 | 不出现“全网第一”“必买爆款”等绝对化表达 |

### 2.3 微信机器人发送

| 用例 | 测试点 | 预期结果 |
| --- | --- | --- |
| 定时发送 | 每日固定时间触发 | 按设定时间发送成功 |
| 群聊发送 | 私域群推荐 | 发送到指定群，不误发其他群 |
| 图片和链接发送 | 推荐消息发送 | 先发送一图流卡片，再发送产品链接 |
| 公域内容生成 | 公域渠道内容 | 使用短文案，不发送私域长文案 |
| 发送失败重试 | 微信接口异常 | 有重试机制，失败后记录日志 |
| 权限控制 | 非授权群触发 | 不发送推荐内容 |
| 一键暂停 | 管理员发送暂停指令 | 当前群停止自动推荐 |

### 2.4 防刷屏

| 用例 | 测试点 | 预期结果 |
| --- | --- | --- |
| 单群每日推荐次数 | 同一群重复触发 | 不超过设定上限 |
| 连续消息控制 | 多条商品推荐 | 合并发送或分批发送，有间隔 |
| 用户频繁请求 | 多用户连续触发 | 触发限流或排队 |
| 异常循环发送 | 任务重复执行 | 系统自动中断并告警 |

## 3. 验收标准

### 3.1 推荐质量

- 饰品、玩具分类准确率 >= 95%。
- 推荐商品至少命中一个有效趋势来源。
- 高优先级推荐商品优先来自多来源交叉验证。
- 过期热点、过期销量数据不得进入核心推荐池。
- 同一商品 3 天内不重复推送给同一群。

### 3.2 内容质量

- 私域推荐内容完整、清晰、可读。
- 公域推荐内容简洁，不暴露内部信息。
- 一图流卡片必须展示产品主图、产品名称、火热指数或卖点、来源信息。
- 每条推荐必须带产品链接；链接不可用时不得自动推送。
- 图片文字不得重叠，主图不得严重裁切，手机端预览清晰可读。
- 文案不得包含明显虚假、夸大、违法违规表述。
- 同一商品连续推荐不得重复文案超过 2 天。
- 玩具类必须保留安全和年龄适用风险提示字段。

### 3.3 系统稳定性

- 每日任务执行成功率 >= 99%。
- 消息发送失败可追踪、可重试。
- 所有推荐结果、发送记录、审核记录均有日志。
- 出现异常时可暂停自动发送。

### 3.4 运营可控

- 支持人工审核后发送。
- 支持黑名单商品、品牌、关键词。
- 支持指定群开启 / 关闭推荐。
- 支持一键暂停机器人推荐任务。

## 4. 数据质量检查

### 4.1 数据来源检查

- 社媒热点数据必须标记来源、时间、关键词、热度值。
- 电商销量数据必须标记平台、品类、7 日销量变化、增长率。
- 国际时尚大会 / 杂志数据必须标记来源名称、发布时间、趋势关键词。
- 数据时间窗口必须严格校验：
  - 社媒：3 日内。
  - 电商：7 日内。
  - 时尚趋势：MVP 建议 30 日内。

### 4.2 商品数据检查

- 商品必须包含名称、类目、链接、产品主图、价格区间、卖点。
- 商品类目必须属于饰品或玩具。
- 去重同款、相似款、重复链接。
- 过滤无图、无价格、无来源、标题异常、疑似违禁商品。

### 4.3 推荐权重检查

- 权重计算必须可解释。
- 每个推荐商品应展示推荐依据。
- 单一来源异常暴涨不得直接成为最高推荐。
- 权重配置变更需要记录版本。

## 5. 内容合规和反刷屏策略

### 5.1 内容合规

禁止使用：

- 绝对化用语：最强、第一、全网最低、必买、稳赚。
- 虚假承诺：保值、升值、治疗、益智保证、100% 安全。
- 诱导分享、诱导转发、诱导加群话术。
- 疑似侵权 IP、仿冒品牌、盗版潮玩。
- 敏感、暴力、成人化、低俗内容。

要求：

- 明确区分“趋势推荐”和“销售承诺”。
- 玩具类注意儿童安全和年龄适用。
- 饰品类注意材质和过敏风险。

### 5.2 反刷屏

- 每个私域群每日推荐不超过 1-2 次。
- 单次推荐商品数量建议 3-5 个。
- 长内容合并为一条或结构化卡片。
- 同一商品至少间隔 3 天再次推荐。
- 同一群 10 分钟内最多响应 1 次推荐请求。
- 异常发送量超过阈值自动暂停。
- 支持群主 / 管理员关键词关闭：暂停推荐、今日不推。

## 6. 上线前检查清单

### 开源依赖检查

- 已列出所有第三方项目、版本、许可证和用途。
- 已检查 AstrBot、Wechaty、NoneBot2 等参考项目的许可证影响。
- 未直接复制许可证不兼容的代码。
- 依赖版本已锁定或记录。
- 已配置代码格式化、静态检查和测试命令。

### 配置检查

- 已配置目标微信群白名单。
- 已配置公域 / 私域不同文案模板。
- 已配置饰品、玩具类目映射。
- 已配置推荐权重规则。
- 已配置敏感词、违禁词、黑名单商品。
- 已配置每日发送时间和发送上限。

### 数据检查

- 最近 3 日社媒热点数据可用。
- 最近 7 日电商销量增长数据可用。
- 时尚大会 / 杂志趋势数据可用。
- 商品链接、图片、价格、类目完整。
- 测试样本中无明显错类商品。

### 功能检查

- 定时任务正常。
- 手动触发正常。
- 私域长文案生成正常。
- 公域短文案生成正常。
- 一图流卡片生成正常。
- 图片卡片和产品链接发送顺序正常。
- 微信发送正常。
- 失败重试正常。
- 日志记录正常。
- 人工审核流程可用。
- 一键暂停功能可用。

### 安全检查

- 不向非授权群发送。
- 不暴露内部权重、接口、客户信息。
- 不记录敏感个人隐私。
- 机器人异常时不会无限发送。
- 有管理员通知和异常告警。

## 7. MVP 监控指标

| 类型 | 指标 |
| --- | --- |
| 推荐效果 | 推荐成功生成率、点击率、互动率、转化率、人工驳回率、负反馈率 |
| 数据质量 | 数据源可用率、类目识别准确率、重复商品比例、过期数据占比、无效链接比例 |
| 内容质量 | 合规通过率、敏感词命中率、夸大宣传拦截次数、文案错用次数 |
| 图片卡片 | 卡片生成成功率、主图可用率、图片发送成功率、链接有效率 |
| 机器人稳定性 | 每日任务执行成功率、消息发送成功率、平均发送延迟、接口失败次数、重试次数 |
| 反刷屏 | 单群每日发送次数、单用户触发频率、限流触发次数、重复推荐次数 |

## 8. 灰度验收建议

- 连续试运行 7 天。
- 每天人工抽检不少于 20 条推荐结果。
- 私域群无明显刷屏投诉。
- 推荐商品类目准确率达到 95% 以上。
- 内容合规通过率达到 98% 以上。
- 每日任务稳定执行，无重复群发事故。
