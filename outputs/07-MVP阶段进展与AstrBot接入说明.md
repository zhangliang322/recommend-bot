# MVP 阶段进展与 AstrBot 接入说明

更新时间：2026-07-13

## 本阶段已完成

- 建立统一每日推荐流水线：采集、类目过滤、评分、去重、安全检查、卡片生成。
- AstrBot 插件入口已按当前官方接口实现，插件主文件为 `main.py`。
- `/今日精品`：生成推荐卡片，并按“产品主图卡片 + 产品链接”发送。
- `/登记推荐群`：保存当前群聊的统一会话标识，为主动推送做准备。
- `/推荐群状态`：检查目标群是否登记成功。
- 命令仅限 AstrBot 管理员调用，避免普通群成员误触发批量推荐。
- 群目标持久化保存，机器人重启后仍可读取。

## 验证结果

- 自动化测试：9 项全部通过。
- 本地完整演示：成功生成 4 个候选推荐和 4 张 PNG 卡片。
- Python 语法与模块编译检查：通过。
- 当前发送仍使用示例商品和模拟采购链接，尚未对真实微信群产生外部发送。

## AstrBot 测试步骤

1. 在 AstrBot 使用的 Python 环境中安装本项目：`pip install -e <项目根目录>`。
2. 把 `integrations/astrbot_plugin_product_reco` 放入 AstrBot 的 `data/plugins/`。
3. 设置 `PRODUCT_RECO_PROJECT_ROOT` 为本项目根目录，然后启动或重载 AstrBot。
4. 在内部测试群中，由管理员依次发送 `/登记推荐群`、`/推荐群状态`、`/今日精品`。
5. 确认每个推荐均收到一张产品卡片和紧随其后的产品链接。

## 下一开发节点

1. 选择并接通实际微信通道。优先验证企业微信或 AstrBot 当前稳定支持的微信适配器。
2. 增加每日定时任务，读取已登记群会话并主动发送。
3. 将 CSV 采集器替换为可配置数据源：TikTok、Instagram、小红书、电商接口。
4. 加入人工审核队列，只有已批准商品可以进入正式群推送。
5. 将示例采购链接替换为真实供应链链接，并增加库存、价格和失效检查。

## 当前边界

社媒和电商平台容易发生接口、登录和风控变化。MVP 保持统一采集器接口，后续可以替换开源项目或官方 API，而不修改评分、卡片和 AstrBot 插件层。真实采集上线前仍需确认平台条款、账号授权方式和请求频率。
