# 自动推送与人工审核阶段交付

更新时间：2026-07-13

## 已完成能力

- 使用成熟的 APScheduler 执行每日定时任务。
- 支持 `Asia/Shanghai` 时区和可配置推送时间。
- 自动推送目标来自管理员登记的 AstrBot 统一会话标识。
- 同一群同一天成功发送后写入台账，防止机器人重启或任务重复执行造成二次推送。
- 自动推送具备双开关：只有 `auto_push_enabled: true` 且 `dry_run: false` 才会发送。
- 增加商品人工批准清单，自动任务只发送明确批准的商品。
- 插件停止或重载时会正确关闭调度器。

## 管理员指令

| 指令 | 用途 |
| --- | --- |
| `/今日精品` | 在当前会话预览推荐卡片和产品链接 |
| `/登记推荐群` | 将当前会话登记为自动推荐目标 |
| `/推荐群状态` | 查看已登记的目标群 |
| `/待审商品` | 查看候选商品、热度分和批准状态 |
| `/批准商品 商品ID` | 允许商品进入自动推送 |
| `/撤销批准 商品ID` | 撤销商品的自动推送资格 |
| `/自动推荐状态` | 查看开关、时间、目标群和已批准商品数量 |

## 灰度启用顺序

1. 在 AstrBot 中接入一个内部微信测试群。
2. 管理员发送 `/登记推荐群`。
3. 使用 `/今日精品` 验证卡片和链接展示。
4. 使用 `/待审商品` 查看候选，并批准 1 个测试商品。
5. 保持 `dry_run: true`，先确认 `/自动推荐状态`。
6. 确认群和商品均无误后，将 `auto_push_enabled` 改为 `true`、`dry_run` 改为 `false`。
7. 重载插件，使新的推送时间和开关生效。
