# 微信私域精品推荐机器人 MVP 产品需求文档

## 1. 产品定位

开发一个基于微信机器人的每日精品产品推荐工具，面向私域电商运营、跨境选品、供应链商家和内容运营人员，自动或半自动筛选近期有热度、有采购可能、有内容传播价值的商品，并生成适合私域群和公域渠道使用的推荐内容。

MVP 阶段优先测试两个垂类：

- 饰品：耳环、项链、手链、戒指、发饰、挂件等。
- 玩具：潮玩、毛绒、解压玩具、益智玩具、儿童玩具、收藏玩具等。

机器人框架倾向参考 AstrBot 的插件式思路：微信消息、群指令、定时任务和平台适配由机器人层处理；商品推荐、评分、文案、数据源接入作为独立业务模块，避免后续换机器人框架时重写核心逻辑。

## 2. 目标用户

| 用户 | 核心需求 |
| --- | --- |
| 私域电商运营 | 每天稳定获得可发到微信群、朋友圈、视频号的新品或爆品推荐 |
| 跨境选品 / 采购人员 | 快速发现饰品、玩具类近期趋势商品和采购入口 |
| 小型品牌 / 供应链商家 | 用热点趋势包装商品卖点，提高私域转化 |
| 内容运营 / 带货文案人员 | 快速生成不同渠道可用的推荐文案 |

## 3. 核心场景

1. 每日自动推送
   - 机器人每天固定时间向指定私域群推送精品产品推荐。

2. 群内手动触发
   - 运营人员在微信群输入“今日推荐”“饰品推荐”“玩具推荐”“爆款榜”等关键词，机器人返回推荐结果。

3. 私域深度推荐
   - 在内部运营群或客户群内输出产品来源、采购链接、推荐理由、火热指数、适合人群、风险提示。

4. 公域轻量种草
   - 生成适合朋友圈、小红书、视频号、TikTok 等渠道使用的产品文案，只展示产品和卖点，不暴露采购来源、成本价和内部评分逻辑。

5. 人工反馈沉淀
   - 运营可以对推荐结果标记“好”“一般”“差”“已采购”“屏蔽”，用于后续调权。

## 4. MVP 范围

### 4.1 必须实现

- 微信机器人基础接入。
- 指定微信群白名单。
- 每日定时推送。
- 群内关键词触发推荐。
- 饰品、玩具两个垂类。
- 手动导入趋势 / 商品数据。
- 规则评分生成火热指数。
- 私域详细推荐卡片。
- 一图流推荐卡片：产品主图 + 核心推荐信息 + 产品链接。
- 公域短文案生成。
- 推荐日志记录。
- 一键暂停 / 群内暂停指令。

### 4.2 推荐权重来源

MVP 推荐权重来自三类信号：

| 信号 | 时间窗口 | 说明 |
| --- | --- | --- |
| 社交媒体热点 | 3 日内 | TikTok、Instagram、小红书的热点话题、关键词、互动增长 |
| 电商销量增长 | 7 日内 | 近期销量、榜单、评论、收藏、排名快速增长的商品 |
| 国际时尚趋势 | 建议 30 日内 | 国际时尚大会、Vogue、Elle、Harper's Bazaar、WWD、Pinterest Trends 等趋势关键词 |

MVP 不强依赖全自动抓取。第一版建议采用“人工整理数据表 + 少量可接入数据源 + 规则评分”的方式跑通闭环。

## 5. 暂不做范围

- 不做完整自动爬虫系统。
- 不做自动下单、库存同步、支付、物流。
- 不做客户级千人千面推荐。
- 不同时支持 WhatsApp、Telegram、企业微信等多平台。
- 不做完整 BI 看板。
- 不自动发布到公域平台。
- 不承诺高精度爆款预测。

## 6. 用户流程

1. 运营配置目标微信群、推荐类目、推送时间、推荐数量、公域 / 私域模式。
2. 每日导入或采集趋势数据、商品数据、采购链接。
3. 系统进行商品标准化、去重、标签匹配。
4. 系统按规则计算火热指数。
5. 系统生成一图流推荐卡片、公域文案和产品链接消息。
6. 机器人先向管理员推送预览。
7. 管理员确认后自动或手动推送到指定群。
8. 运营在群内反馈推荐质量。
9. 系统记录反馈，用于后续调权。

## 7. 最终呈现形式

MVP 最终呈现优先采用“一图流”：

- 第一条：产品推荐图片卡片，包含产品主图和核心推荐信息。
- 第二条：产品链接，优先发送采购链接或商品详情链接。
- 可选第三条：极短补充说明，控制在 80 字以内。

一图流卡片建议布局：

| 区域 | 内容 |
| --- | --- |
| 顶部 | 类目、火热指数、推荐等级 |
| 主视觉 | 产品主图，必须清晰、无遮挡 |
| 中部 | 产品名称、一句话卖点 |
| 底部 | 推荐理由 2-3 点、来源标识、风险提示短句 |
| 角标 | 私域 / 公域版本标识，可选 |

微信群内不建议直接发送大段文字。长字段仍保存在系统详情页或 `/reco detail 商品ID` 中，群内只展示图片卡片和链接。

## 8. 私域推荐卡片字段

| 字段 | 说明 |
| --- | --- |
| 产品名称 | 如“蝴蝶结珍珠发夹”“解压毛绒挂件” |
| 类目 | 饰品 / 玩具 |
| 产品主图 | 必填，用于生成一图流推荐卡片 |
| 产品来源 | TikTok / Instagram / 小红书 / Amazon / Temu / 1688 / 展会 / 杂志等 |
| 热点来源说明 | 说明近期热度来自哪里 |
| 采购链接 | 1688、拼多多、速卖通、供应商链接等 |
| 参考售价 | 可选，用于判断价格带 |
| 采购价 / 成本价 | 可选，仅私域内部使用 |
| 推荐理由 | 说明为什么值得推 |
| 火热指数 | 0-100 分 |
| 指数构成 | 社媒热度、电商增长、趋势匹配、采购可得性 |
| 推荐人群 | 如年轻女性、学生党、亲子家庭、潮玩用户 |
| 推荐话术 | 可直接复制到私域群 |
| 风险提示 | 如材质、年龄适用、侵权风险、同款竞争 |
| 卡片图片 URL | 系统生成的一图流成品图地址 |

## 9. 公域推荐字段

- 产品名称。
- 一句话卖点。
- 种草文案。
- 适合发布平台。
- 推荐标签。
- 产品主图或一图流公域版卡片。

公域不展示采购链接、成本价、供应商信息、内部火热指数构成。

## 10. 优先级

### P0：MVP 闭环

- 微信机器人定时推送。
- 群内关键词触发推荐。
- 饰品 / 玩具类目。
- 手动导入数据。
- 规则打分。
- 私域推荐卡片。
- 一图流卡片生成。
- 产品链接随卡片发送。
- 公域文案模式。
- 基础配置。
- 推荐日志。
- 防刷屏和暂停能力。

### P1：增强可用性

- 图片展示。
- 人工反馈指令。
- 不同群配置不同类目。
- 每日推荐去重。
- 采购链接白名单。
- 文案风格选择：种草、促销、专业选品。

### P2：后续优化

- 自动抓取社媒和电商趋势。
- 接入第三方商品数据 API。
- 个性化推荐。
- 推荐效果追踪。
- Web 管理后台。
- 多渠道发布。
- 更复杂的趋势预测模型。

## 11. MVP 成功标准

- 连续 7 天每日稳定向测试群推送推荐。
- 每天生成 6-10 个可用产品推荐。
- 每条推荐至少包含 1 张产品主图卡片和 1 个产品链接。
- 运营可直接复制公域文案发布。
- 私域卡片能支持采购判断。
- 至少 30% 推荐被运营标记为“可跟进”或“已采用”。
- 无误发群、刷屏、无限循环发送事故。
