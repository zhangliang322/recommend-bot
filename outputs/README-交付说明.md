# 微信私域精品推荐机器人 MVP 文档交付说明

本次交付包含四份文件：

1. `01-产品需求文档-微信私域精品推荐机器人MVP.md`
   - 用于确认产品目标、MVP 范围、用户流程、推荐卡片字段和成功标准。

2. `02-开发节点文档-微信私域精品推荐机器人MVP.md`
   - 用于指导开发，包括模块划分、数据源策略、评分公式、存储结构、机器人指令、定时任务和里程碑。

3. `03-测试验收文档-微信私域精品推荐机器人MVP.md`
   - 用于测试和上线审核，包括测试用例、验收标准、数据质量、内容合规、防刷屏和监控指标。

4. `04-MVP执行清单-微信私域精品推荐机器人.md`
   - 用于最快跑通第一版，包含第一周任务、样本数据字段、模板、最小指令集和灰度测试安排。

5. `05-开源项目参考与工程规范.md`
   - 用于约束项目不要从零开发，优先参考成熟开源项目，并明确代码、文档、许可证规范。

6. `06-数据源复用方案-社媒与电商.md`
   - 用于约束 TikTok、Instagram、小红书和电商平台的数据接入优先复用现成项目、官方 API、联盟 API 和第三方接口。

建议下一步先按 `04-MVP执行清单` 准备 20-40 条饰品 / 玩具样本数据，然后进入插件骨架和评分模块开发。

补充决策：推荐最终呈现优先采用“一图流产品推荐卡片 + 产品链接”。长字段保留在详情指令或后台数据中，微信群内只发图片卡片和链接，降低阅读成本和刷屏风险。

补充工程原则：项目构建采用“成熟开源项目参考 + 模块化拼接”，优先复用 AstrBot、Wechaty、NoneBot2、APScheduler、Pydantic、SQLAlchemy、Pillow、pytest、Ruff 等成熟生态能力。

补充数据源原则：TikTok、Instagram、小红书和电商数据不从零开发采集器，优先复用 TikTok-Api、Instaloader、MediaCrawler、Crawlee、Scrapy、SerpApi、官方开放平台和联盟 API。
