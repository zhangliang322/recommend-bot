# 数据源复用方案：社媒与电商

## 1. 目标

TikTok、Instagram、小红书和电商平台的数据接入优先复用 GitHub 上成熟项目、官方 API、联盟 API 和第三方接口，不从零开发采集器。

MVP 仍保留 CSV / 表格导入，保证自动采集失败时推荐流程不断。

## 2. 统一采集接口

所有数据源以适配器形式接入：

```text
CollectorAdapter
  name
  platform
  supported_modes
  collect_keywords(keywords, since, limit)
  collect_product(product_url_or_id)
  normalize(raw_item)
  health_check()
```

统一输出字段：

```text
source_platform
source_type
keyword
title
description
content_url
image_url
product_url
publish_time
collected_at
metrics_json
raw_json
terms_note
```

## 3. 社媒数据源

| 平台 | 候选项目 / 接口 | 适合采什么 | MVP 建议 |
| --- | --- | --- | --- |
| TikTok | https://github.com/davidteather/TikTok-Api | 趋势视频、公开视频、用户公开信息 | 可做第一批自动采集候选 |
| TikTok | TikHub / EnsembleData / Apify Actor | 视频、用户、评论、趋势、商品 | 生产稳定性兜底，有成本 |
| Instagram | https://github.com/instaloader/instaloader | 图片、视频、caption、metadata、公开主页 | 可采集标签和账号素材 |
| Instagram | Apify Actor / SerpApi | 搜索结果、帖子、图片 | 生产兜底 |
| 小红书 | https://github.com/NanmiCoder/MediaCrawler | 小红书关键词、笔记、评论等公开信息 | 适合评估，但需重点看合规和稳定性 |
| 多平台 | https://github.com/apify/crawlee | 自建采集器，代理、会话、Playwright / Puppeteer | 第二阶段使用 |

## 5. 推荐接入顺序

第一阶段：

- CSV 导入。
- TikTok-Api 评估。
- Instaloader 评估。
- MediaCrawler 评估。
- SerpApi Shopping 或人工榜单导入。

第二阶段：

- 官方联盟 API：淘宝联盟、多多进宝、AliExpress Affiliate。
- 第三方 API：TikHub、EnsembleData、Apify。
- Crawlee / Scrapy 自建稳定采集器。

第三阶段：

- 数据源健康评分。
- 自动切换备用采集器。
- 按采集成本、成功率、数据质量动态调度。

## 6. 风险和约束

- 非官方采集器容易因平台改版失效。
- 社媒平台可能限制自动化访问，必须限流和保留人工兜底。
- 登录态、cookie、token 不得写入代码或日志。
- 不采集私密内容、私信、个人隐私字段。
- 所有第三方项目和 API 都必须记录到 `THIRD_PARTY_NOTICES.md`。
- 采集器异常不能影响评分、卡片生成和微信推送主流程。
