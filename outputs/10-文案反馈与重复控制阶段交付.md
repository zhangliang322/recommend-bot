# 文案、反馈与重复控制阶段交付

更新时间：2026-07-13

## 已完成能力

- 生成私域详细推荐：商品来源、采购链接、火热指数、目标客户、推荐理由和风险提示。
- 生成公域精简文案：产品名称、卖点、适用人群、参考价格、产品链接和话题标签。
- 支持按“饰品”或“玩具”查看最多 3 个推荐卡片。
- 支持运营人员对商品记录“好、一般、差”评价和备注。
- 正式自动推送成功后记录商品、目标群和发送时间。
- 推荐评分已接入历史新鲜度，降低短期重复推荐概率。

## 新增 AstrBot 指令

| 指令 | 功能 |
| --- | --- |
| `/商品详情 商品ID` | 查看私域完整推荐依据 |
| `/公域文案 商品ID` | 生成可复制到公域渠道的精简文案 |
| `/分类推荐 饰品` | 查看饰品推荐卡片 |
| `/分类推荐 玩具` | 查看玩具推荐卡片 |
| `/反馈商品 商品ID 好` | 记录正向反馈 |
| `/反馈商品 商品ID 一般 备注` | 记录一般反馈及备注 |
| `/反馈商品 商品ID 差 备注` | 记录负向反馈及备注 |

## 新鲜度规则

| 距上次正式推荐 | 新鲜度分 |
| --- | ---: |
| 7 天以内 | 20 |
| 7–14 天 | 60 |
| 14 天以上或从未推荐 | 100 |

新鲜度占综合火热指数的 5%。它不会简单屏蔽高热商品，但会让同等条件下的新商品优先进入推荐池。

## 数据记录

- `recommendation_history.jsonl`：正式发送商品、目标群和发送时间。
- 所有运行数据保存在 AstrBot 插件数据目录，不写入源码和配置文件。
